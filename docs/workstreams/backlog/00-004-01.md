# 00-004-01: Eliminate Circular Dependencies

Feature: F004 (Resource Economy System Integration)

## Goal

Remove circular import dependencies between `economy` and `simulation` packages to enable integration. Ensure both packages can be built independently and conversion functions work correctly.

## Context

The `economy` package currently imports `simulation` in several test files and in `seasons.go`. The `simulation` package needs to import `economy` for integration, creating a Go import cycle. This workstream breaks the cycle by:
1. Removing unused imports of `simulation` from `economy` non‑test code
2. Updating test files to use the `economy/conversion` package instead of direct `simulation` imports where possible
3. Verifying that both packages compile independently

## Dependencies

- Independent (prerequisite for all other F004 workstreams)

## Acceptance Criteria

- [ ] `internal/economy/seasons.go` no longer imports `simulation` (import is unused)
- [ ] All economy test files that need `simulation` types use `economy/conversion` package for conversions
- [ ] `go build ./internal/economy` succeeds without import cycle errors
- [ ] `go build ./internal/simulation` succeeds (simulation may still import economy, but no cycle)
- [ ] `go test ./internal/economy/... -short` passes (tests may be updated but must remain green)
- [ ] The `economy/conversion` package provides `ToSimulationResource` and `FromSimulationResource` functions
- [ ] No regression in existing test coverage

## Steps

1. **Remove unused import from seasons.go**
   - Check if `simulation` import is actually used in `seasons.go` (it appears unused)
   - Remove the import line
   - Verify file still compiles

2. **Update inventory_test.go**
   - Already uses `conversion` package for `LoadInventoryFromGameState` and `ExportInventoryToGameState`
   - Ensure test passes with the updated import

3. **Update other test files that import simulation**
   - `seasons_test.go`, `types_test.go`, `benchmark_test.go`, `recipes_test.go`, `spolage_test.go`, `scarcity_test.go`
   - For each file, check if the import is needed for types or functions
   - If only conversion functions are needed, switch to `economy/conversion`
   - Keep `simulation` import if direct type references are required (e.g., `simulation.Resource`)

4. **Verify conversion package completeness**
   - Ensure `conversion` package exports `ToSimulationResource` and `FromSimulationResource`
   - Add any missing conversion helpers

5. **Run verification commands**
   - Execute `go build ./internal/economy`
   - Execute `go build ./internal/simulation`
   - Run economy tests with `-short` flag

## Code

Helper snippet for checking import usage:
```bash
grep -n "simulation\." internal/economy/seasons.go
```

If no matches, the import can be safely removed.
