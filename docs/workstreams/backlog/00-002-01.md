# 00-002-01: Resource Definition System

Feature: F002 (Resource Economy System)

## Goal

Define the 15 core resource types and their properties for the medieval economy simulation. Create data structures for resources with quality tiers, spoilage rates, and base values. Establish constants for all resource types (grain, vegetables, wood, stone, iron ore, wool, flour, bread, planks, iron, cloth, tools, furniture, weapons, clothing) and their categorization (raw, processed, advanced).

## Scope Files

- `internal/economy/types.go` - Resource type definitions, structs, interfaces
- `internal/economy/constants.go` - Constants for resource types, categories, properties
- `internal/economy/resources.go` - Resource management functions (creation, validation)
- `internal/economy/types_test.go` - Unit tests for resource definitions

## Dependencies

None (first workstream for F002)

## Acceptance Criteria

- [ ] 15 resource types defined as constants with proper categorization
- [ ] Resource struct with fields: Type, Quantity (float64), Quality (QualityTier), Location, Produced (GameDate), Value
- [ ] QualityTier enum: Poor, Normal, Good, Excellent, Masterwork (0-4)
- [ ] Resource categories: Raw (6), Processed (5), Advanced (4)
- [ ] Each resource type has base spoilage rate (food spoils faster than materials)
- [ ] Resource validation functions (IsValidType, IsPerishable, GetCategory)
- [ ] Unit tests covering all resource types and validation logic
- [ ] `go build ./internal/economy` passes
- [ ] `go test ./internal/economy -v` passes with ≥80% coverage
- [ ] Integration with existing simulation types (backward compatibility)
