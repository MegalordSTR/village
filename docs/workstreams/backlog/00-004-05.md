# 00-004-05: Update Production System to Use Economy Types

Feature: F004 (Resource Economy System Integration)

## Goal

Update the production system (`production.go`) to use economy resource types (`economy.ResourceType`) and quality tiers (`economy.QualityTier`) instead of string literals. Ensure produced resources are created with proper economy types and can be stored in the inventory.

## Context

The production system currently uses string resource types ("wood", "grain", "tools") and float64 quality values. The economy package defines enumerated resource types and discrete quality tiers. This workstream aligns production with economy types, so that produced resources are compatible with the inventory system and can benefit from economy features (spoilage, value calculation, storage restrictions).

## Dependencies

- WS-004-02: Update Simulation Types to Use Economy Types (ensures `Production` struct uses `economy.ResourceType`)

## Acceptance Criteria

- [ ] Production functions use `economy.ResourceType` constants instead of string literals
- [ ] Resource quality is expressed as `economy.QualityTier` (or converted from float64)
- [ ] Produced resources are added to `GameState.Inventory` (if present) with appropriate location and production date
- [ ] Existing production tests pass (`go test ./internal/simulation -run TestProduction`)
- [ ] `go build ./internal/simulation` passes
- [ ] Deterministic production behavior maintained (same seed yields same output)

## Steps

1. **Update `Production` struct usage**
   - Ensure `Production.ResourceType` is already `economy.ResourceType` (from WS‑004‑02)
   - Update any remaining string literals in production.go to use economy constants (e.g., `"wood"` → `economy.ResourceWood`)

2. **Refactor `produceResources` function**
   - Instead of creating `simulation.Resource` directly, create `economy.Resource` with proper fields:
     - `Type`: from production plan
     - `Quantity`: calculated amount
     - `Quality`: convert from float64 to `QualityTier` (use `economy.FloatToQuality`)
     - `Location`: assign based on producing building (e.g., building ID)
     - `Produced`: set to current game date (from `GameState.Calendar`)
     - `Value`: can be left as `economy.BaseValue(type)`
   - Add the resource to `GameState.Inventory` if present; otherwise fall back to `GameState.Resources` (via sync)

3. **Update resource consumption in production chains**
   - When a production chain consumes raw materials (e.g., wood to make planks), use `Inventory.RemoveResource` if inventory exists
   - Map legacy resource names to economy types (e.g., "ore" → `economy.ResourceIronOre`)

4. **Handle building‑specific production**
   - Farms produce `ResourceGrain` or `ResourceVegetables`
   - Mines produce `ResourceStone` or `ResourceIronOre`
   - Workshops produce `ResourceTools`, `ResourceFurniture`, etc.
   - Ensure each building type maps to correct economy resource types

5. **Update quality calculation**
   - Production quality depends on worker skills, building level, and random factors
   - Convert final quality float (0.0–1.0) to `QualityTier` using `economy.FloatToQuality`
   - Ensure extreme values (0.0 → `QualityPoor`, 1.0 → `QualityMasterwork`) map correctly

6. **Update tests**
   - Tests that check produced resource types need updated assertions (compare with `economy.ResourceType` constants)
   - Tests that verify quality may need tolerance adjustments (float vs tier)
   - Ensure tests still pass with both old and new code paths (backward compatibility)

7. **Verification**
   - Build simulation package
   - Run production‑related tests
   - Optionally add a small integration test that produces a resource, verifies it appears in inventory with correct fields

## Code

Example conversion in `produceResources`:
```go
// Convert simulation production to economy resource
rt := economy.ResourceType(prod.ResourceType)
qualityTier := economy.FloatToQuality(prod.Quality)
r := economy.Resource{
    Type:     rt,
    Quantity: float64(prod.Amount),
    Quality:  qualityTier,
    Location: building.ID,
    Produced: economy.GameDate{Year: state.Calendar.Year, Week: state.Calendar.Week},
    Value:    economy.BaseValue(rt),
}
// Add to inventory if available
if state.Inventory != nil {
    state.Inventory.AddResource(building.ID, r)
} else {
    // Fallback to legacy slice
    state.Resources = append(state.Resources, FromEconomyResource(r))
}
```
