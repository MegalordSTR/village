# 00-004-11: Add Observability (Logging/Metrics) for Inventory Operations

Feature: F004 (Resource Economy System Integration)

## Goal

Introduce logging and metrics for inventory operations to improve debuggability and monitoring. After consolidation, inventory becomes the sole resource storage; we need visibility into its behavior, especially for sync operations (now removed) and error conditions.

## Context

While the dualŌĆæstorage sync is being removed, inventory operations themselves still lack visibility. Logging will help diagnose issues like validation failures, capacity violations, and unexpected consumption patterns.

## Code

Snippet for logging in `AddResource`:

```go
import "log"

func (inv *Inventory) AddResource(location string, r Resource) error {
    if !r.Validate() {
        log.Printf("WARN: invalid resource attempted to add at %s: type=%s quantity=%f", location, r.Type, r.Quantity)
        return errors.New("invalid resource")
    }
    if inv.storage != nil {
        if !inv.storage.CanStoreResourceAt(location, r.Type) {
            log.Printf("WARN: resource type %s cannot be stored at %s", r.Type, location)
            return errors.New("resource type cannot be stored at this location")
        }
        available := inv.storage.AvailableCapacity(location, inv)
        if r.Quantity > available {
            log.Printf("WARN: insufficient capacity at %s for %s (need %f, available %f)", location, r.Type, r.Quantity, available)
            return errors.New("insufficient storage capacity")
        }
    }
    inv.resources[location] = append(inv.resources[location], r)
    log.Printf("INFO: added resource %s quantity %f at %s", r.Type, r.Quantity, location)
    return nil
}
```
