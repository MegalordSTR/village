# 00-004-06: Update Tests and Ensure Backward Compatibility

Feature: F004 (Resource Economy System Integration)

## Goal

Update all simulation tests to work with the integrated economy system, verify backward compatibility (existing save files load correctly), and ensure deterministic behavior is preserved. Run full test suite to confirm no regressions.

## Context

After updating simulation types, inventory integration, economic system, and production system, many tests will fail because they rely on legacy resource types ("food", "wood") and direct manipulation of `GameState.Resources`. This workstream updates those tests to use economy types and inventory‑aware helpers, while maintaining the same assertions about behavior. It also adds integration tests that verify the synchronization between `Resources` and `Inventory` works correctly.

## Code

Example test helper for deterministic verification:
```go
func TestDeterministicWithInventory(t *testing.T) {
    seed := int64(12345)
    state1 := simulation.NewGameState("test1", seed)
    state2 := simulation.NewGameState("test2", seed)
    // Add same resources to both states (using legacy Resources slice)
    state1.AddResource(simulation.Resource{Type: "grain", Quantity: 100, Quality: 0.5})
    state2.AddResource(simulation.Resource{Type: "grain", Quantity: 100, Quality: 0.5})
    // Sync inventory so economic system can use it
    state1.SyncInventory()
    state2.SyncInventory()
    // Run economic update for 10 weeks
    econ := simulation.NewEconomicSystem()
    for i := 0; i < 10; i++ {
        econ.Update(i, state1, state1.RNG.Rand())
        econ.Update(i, state2, state2.RNG.Rand())
    }
    // Both states should have identical Resources and Inventory totals
    state1.SyncResources()
    state2.SyncResources()
    if !reflect.DeepEqual(state1.Resources, state2.Resources) {
        t.Errorf("determinism broken: Resources differ")
    }
}
```
