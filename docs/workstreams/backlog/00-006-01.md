# 00-006-01: Update Go Version and CI Configuration

Feature: F006 (Go Modernization)

## Goal

Ensure consistent use of Go 1.25.7 across all project configurations (CI/CD, Docker, go.mod). Update GitHub Actions workflows to use the same major version as specified in go.mod to prevent compatibility issues.

## Scope Files

- `.github/workflows/go.yml` - Update go-version from '1.23' to '1.25'
- `.github/workflows/deploy.yml` - Verify already uses '1.25'
- `Dockerfile` - Ensure uses golang:1.25-alpine
- `go.mod` - Verify contains go 1.25.7
- `Makefile` - Optional: add version check target

## Dependencies

- Independent (infrastructure change)

## Acceptance Criteria

- [ ] `.github/workflows/go.yml` uses `go-version: '1.25'`
- [ ] All GitHub Actions workflows use consistent Go major version (1.25)
- [ ] `Dockerfile` uses `golang:1.25-alpine` base image
- [ ] `go.mod` contains `go 1.25.7`
- [ ] Project builds successfully with Go 1.25.7 locally
- [ ] All tests pass with updated configuration
- [ ] CI workflows execute successfully after change

## Context

Currently, the project's go.mod specifies Go 1.25.7, but the CI workflow (go.yml) uses Go 1.23. This mismatch could lead to compatibility issues where code that works locally fails in CI. The Dockerfile already uses the correct version, and deploy.yml appears to be correctly configured. This workstream ensures consistency across all project configurations.

## Input Files

1. `.github/workflows/go.yml` - Current CI configuration
2. `.github/workflows/deploy.yml` - Deployment workflow
3. `Dockerfile` - Container build configuration
4. `go.mod` - Go module definition
5. `Makefile` - Build automation

## Steps

1. **Update go.yml workflow**
   - Change `go-version: '1.23'` to `go-version: '1.25'` in `.github/workflows/go.yml`
   - Verify no other references to older Go versions exist in the file

2. **Verify deploy.yml workflow**
   - Confirm `.github/workflows/deploy.yml` already uses `go-version: '1.25'`
   - If not, update to match

3. **Check Dockerfile**
   - Ensure `Dockerfile` uses `golang:1.25-alpine` (line 2)
   - Verify no other Go version references in the file

4. **Validate go.mod**
   - Confirm `go.mod` contains `go 1.25.7`
   - If not, update to latest 1.25.x patch version

5. **Optional: Add version check to Makefile**
   - Add target `check-go-version` that validates local Go version matches go.mod
   - Example: `@go version | grep -q "go1.25" || (echo "Go version mismatch" && exit 1)`

6. **Local testing**
   - Run `go version` to confirm local installation is 1.25.x
   - Execute `make test` to ensure tests pass
   - Execute `make build` to ensure project builds

7. **CI validation**
   - Commit changes and push to trigger GitHub Actions
   - Verify all workflows complete successfully
   - Check that go.yml job uses Go 1.25
