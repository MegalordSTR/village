# 00-004-07: Handle Unknown Resource Types in Save File Migration

Feature: F004 (Resource Economy System Integration)

## Goal

Ensure backward compatibility for legacy save files containing unknown resource types by implementing graceful mapping and warning logging. Unknown resource strings are mapped to known economy types with appropriate defaults, preventing crashes and data corruption.

## Context

The `StringToResourceType` function currently maps a limited set of legacy strings ("food", "ore", "tool", "wheat", "gold") to economy constants, defaulting to `ResourceGrain` for any unknown string. This causes silent data loss when save files contain other legacy resource types (e.g., "meat", "wood", "stone"). The QA review identified this as a P1 blocking finding (village‑bpi) because backward compatibility is broken.

Since phase‑4 remediation permits breaking changes, we can also improve the mapping to cover more legacy strings and add logging to alert developers about unmapped types.

## Dependencies

- Independent (can be executed before any other remediation workstreams)

## Code

Example extension of `StringToResourceType`:

```go
import "log"

func StringToResourceType(s string) economy.ResourceType {
    rt := economy.ResourceType(s)
    if economy.IsValidType(rt) {
        return rt
    }
    switch s {
    case "food":
        return economy.ResourceGrain
    case "ore":
        return economy.ResourceIronOre
    case "tool":
        return economy.ResourceTools
    case "wheat":
        return economy.ResourceGrain
    case "gold":
        return economy.ResourceIronOre
    case "meat":
        return economy.ResourceGrain
    case "wood":
        return economy.ResourceWood
    case "stone":
        return economy.ResourceStone
    case "iron":
        return economy.ResourceIron
    case "flour":
        return economy.ResourceFlour
    case "bread":
        return economy.ResourceBread
    // ... add all other legacy strings
    default:
        log.Printf("WARNING: unknown resource type %q mapped to ResourceGrain", s)
        return economy.ResourceGrain
    }
}
```
