# 00-004-03: Integrate Inventory into GameState

Feature: F004 (Resource Economy System Integration)

## Goal

Add `Inventory` field to `GameState` and implement bidirectional synchronization between `GameState.Resources` (legacy slice) and `Inventory` (advanced economy storage). Ensure that existing code using `Resources` continues to work while new economic systems can use `Inventory`.

## Context

The simulation's `GameState` currently stores resources as a simple slice of `Resource`. The economy package provides `Inventory` with location‑based storage, capacity limits, and spoilage tracking. This workstream adds an `Inventory` field to `GameState` and keeps it in sync with the legacy `Resources` slice for backward compatibility.

## Dependencies

- WS-004-02: Update Simulation Types to Use Economy Types

## Acceptance Criteria

- [ ] `GameState` struct has an `Inventory *economy.Inventory` field (tagged `json:"-"` to exclude from serialization)
- [ ] `GameState.SyncInventory()` method initializes `Inventory` from `Resources` if nil
- [ ] `GameState.SyncResources()` method updates `Resources` slice from `Inventory`
- [ ] `AddResource` and `AddBuilding` methods optionally update `Inventory` if present
- [ ] `LoadInventoryFromGameState` and `ExportInventoryToGameState` functions in `economy_integration.go` work correctly
- [ ] Existing tests that manipulate `Resources` still pass (sync happens automatically or tests are updated)
- [ ] `go build ./internal/simulation` passes
- [ ] State‑related tests pass (`go test ./internal/simulation -run "TestState"`)

## Steps

1. **Add Inventory field to GameState**
   - Add `Inventory *economy.Inventory` field with `json:"-"` tag
   - Update `NewGameState` to initialize `Inventory` as `nil` (lazy initialization)

2. **Implement SyncInventory method**
   - If `Inventory` is nil, create a new `economy.Inventory`
   - Call `LoadInventoryFromGameState` to populate it from `Resources`
   - Use a default location (e.g., `"global"`)

3. **Implement SyncResources method**
   - If `Inventory` is not nil, call `ExportInventoryToGameState` and assign to `Resources`
   - This method should be called after any inventory‑modifying operation

4. **Update AddResource and AddBuilding**
   - In `AddResource`, also add to `Inventory` if present (call `SyncInventory` first)
   - In `AddBuilding`, if the building is a storage type, maybe register it with a `StorageRegistry` (future WS)

5. **Ensure conversion functions handle all fields**
   - Verify `LoadInventoryFromGameState` and `ExportInventoryToGameState` in `economy_integration.go` work with updated types
   - Add any missing mapping for location, produced date, etc.

6. **Update existing tests**
   - Tests that directly check `Resources` may need to call `SyncResources` before assertions
   - Alternatively, modify tests to use `Inventory` where appropriate
   - Keep changes minimal; focus on making tests pass

7. **Verification**
   - Build simulation package
   - Run state‑related tests
   - Run a quick integration test that creates a GameState, adds resources, syncs, and verifies both representations match

## Code

Example SyncInventory:
```go
func (gs *GameState) SyncInventory() {
    if gs.Inventory == nil {
        gs.Inventory = economy.NewInventory()
        LoadInventoryFromGameState(gs.Inventory, gs.Resources, "global")
    }
}
```
