# 00-004-09: Remove Unused Conversion Package and Fix Legacy String Comparisons

Feature: F004 (Resource Economy System Integration)

## Goal

Eliminate code duplication by removing the unused `economy/conversion` package and replace all legacy string comparisons (e.g., `"food"`) with proper economy constants, ensuring type safety and consistency.

## Context

TechLead review found an unused `conversion` package that duplicates functions already present in `economy_integration.go` (villageâ€‘g91). Docs review identified a bug where legacy string comparisons like `"food"` may never match `economy.ResourceType` because the constants are different strings (e.g., `"grain"`) (villageâ€‘2a4). The codebase contains numerous direct string literals for resource types, which should be replaced with the appropriate `economy.ResourceType` constants.

Since backward compatibility can be broken, we can safely delete the unused package and update all string literals to use the defined constants.

## Dependencies

- Independent (can be executed anytime)

## Acceptance Criteria

- [ ] The directory `internal/economy/conversion/` is removed entirely

- [ ] All string literals `"food"`, `"ore"`, `"tool"`, `"wheat"`, `"gold"`, `"meat"`, `"wood"`, `"stone"`, `"iron"`, etc. are replaced with the corresponding `economy.ResourceType` constants (`ResourceGrain`, `ResourceIronOre`, `ResourceTools`, â€¦)
- [ ] All comparisons `r.Type == "food"` are changed to `r.Type == economy.ResourceGrain` (or use `IsValidType` where appropriate)

   - Also replace `"grain"` and `"meat"` occurrences (if any) with appropriate constants
   - Update any `switch` statements that use string cases

4. **Replace literals in simulation/social.go**
   - Similar changes for `"food"` and `"grain"` comparisons
