# 00-002-08: Deployment Setup

Feature: F002 (Resource Economy System)

## Goal

Create one-command deployment setup for the full application (Go backend + Angular frontend + PostgreSQL). Configure Docker containers, orchestration with docker-compose, CI/CD pipeline, and Makefile targets for easy development and deployment.

## Scope Files

- `Dockerfile` - Multi-stage build for Go backend and Angular frontend
- `docker-compose.yml` - Services: backend, frontend, postgres, nginx (optional)
- `Makefile` - Development commands (build, test, run, deploy)
- `.github/workflows/deploy.yml` - CI/CD pipeline for automated deployment
- `scripts/deploy.sh` - Deployment script for production

## Dependencies

- 00-002-01 to 00-002-06: All backend systems
- 00-002-07: UI Integration (Angular frontend)

## Acceptance Criteria

- [ ] Dockerfile builds Go backend binary and Angular frontend static files
- [ ] docker-compose.yml defines services: backend (Go API), frontend (nginx serving Angular), postgres (database)
- [ ] Environment variables for configuration (database URL, API port, etc.)
- [ ] Makefile targets: `make build` (builds both backend and frontend), `make test` (runs all tests), `make run` (starts local development), `make deploy` (deploys to production)
- [ ] CI/CD pipeline runs tests, builds Docker images, deploys on push to main branch
- [ ] Deployment script handles database migrations (if needed)
- [ ] Health checks for backend and frontend services
- [ ] Production-ready configuration (security, logging, monitoring)
- [ ] Single command deployment: `make deploy` works locally (simulated) and in CI
- [ ] Documentation in README.md for setup and deployment
- [ ] `docker build -t village-economy .` succeeds
- [ ] `docker-compose config` validates configuration
- [ ] `make deploy-dry-run` shows deployment plan without executing
