# 00-004-02: Update Simulation Types to Use Economy Types

Feature: F004 (Resource Economy System Integration)

## Goal

Update simulation type definitions (`Resource`, `Production`) to use economy package types (`economy.ResourceType`, `economy.QualityTier`) while maintaining backward compatibility through conversion functions.

## Context

The simulation package currently uses its own `Resource` and `Production` types with simple fields (`Type string`, `Quality float64`). The economy package defines richer types with proper constants, categories, and quality tiers. This workstream aligns simulation types with economy types, creating a compatibility layer that allows gradual migration.

## Dependencies

- WS-004-01: Eliminate Circular Dependencies

## Acceptance Criteria

- [ ] `simulation.Resource` type uses `economy.ResourceType` for the `Type` field (or keeps string with validation)
- [ ] `simulation.Resource.Quality` remains `float64` but conversion functions to/from `economy.QualityTier` are available
- [ ] `simulation.Production` uses `economy.ResourceType` for `ResourceType` field
- [ ] Conversion functions `ToEconomyResource` and `FromEconomyResource` are implemented in `economy_integration.go`
- [ ] Existing simulation code that creates `Resource` or `Production` instances continues to work (string literals are automatically converted)
- [ ] `go build ./internal/simulation` passes
- [ ] Relevant type‑related tests pass (`go test ./internal/simulation -run "Test.*Type|Test.*Production"`)

## Steps

1. **Update `types.go`**
   - Change `Resource.Type` from `string` to `economy.ResourceType` (or keep `string` with a comment that it should be validated)
   - Keep `Resource.Quality` as `float64` for backward compatibility
   - Add a helper method `Resource.ToEconomyResource() economy.Resource` if needed
   - Update any constructors or literals in the same file

2. **Update `production.go`**
   - Change `Production.ResourceType` from `string` to `economy.ResourceType`
   - Update any literal strings (e.g., `"wood"`, `"grain"`) to use `economy.ResourceWood`, `economy.ResourceGrain`
   - Ensure production functions still compile

3. **Implement conversion functions in `economy_integration.go`**
   - Ensure `ToEconomyResource` and `FromEconomyResource` handle the updated types
   - Add helper `StringToResourceType` that converts legacy string to `economy.ResourceType` (fallback to `ResourceGrain` for unknown)
   - Add `ResourceTypeToString` for serialization if needed

4. **Update any direct string comparisons in simulation**
   - Search for `r.Type == "food"` or similar patterns in simulation code
   - Replace with `r.Type == economy.ResourceGrain` or use `economy.IsValidType`
   - This may be limited to this workstream; deeper integration will happen in later WS

5. **Run verification**
   - Build simulation package
   - Run type‑ and production‑related tests
   - Ensure no regression in other tests

## Code

Example conversion:
```go
// in economy_integration.go
func StringToResourceType(s string) economy.ResourceType {
    rt := economy.ResourceType(s)
    if economy.IsValidType(rt) {
        return rt
    }
    // Map legacy names
    switch s {
    case "food":
        return economy.ResourceGrain
    case "wood":
        return economy.ResourceWood
    // ... other mappings
    default:
        return economy.ResourceGrain
    }
}
```
