# 00-004-13: Reject Unknown Resource Types and Add JSON Array Size Limits

Feature: F004 (Resource Economy System Integration)

## Goal

Prevent data loss and potential denial-of-service by rejecting unknown resource types with clear errors (instead of silently mapping to a default) and imposing reasonable limits on JSON array sizes during save file loading. This addresses backward compatibility risks and security concerns.

## Context

DevOps review identified a P1 blocking finding (village‑2hn): unknown legacy resource strings are mapped to `ResourceGrain` with a warning log, causing data corruption if legacy saved games contain custom resource types. Security review flagged log spamming (village‑9px, P3) from unknown resource warnings and missing JSON array size limits (village‑x1t, P2) in `LoadInventoryFromGameState`.

Since phase‑4 remediation permits breaking backward compatibility, we can reject unknown types outright and add size limits to prevent memory exhaustion attacks.

## Dependencies

- Independent (can be executed anytime)

## Acceptance Criteria

- [ ] `StringToResourceType` returns an error for unknown strings instead of mapping to `ResourceGrain`
- [ ] `IsKnownType` remains unchanged (still returns `false` for unknown strings)
- [ ] `LoadInventoryFromGameState` rejects unknown resource types with a clear error message
- [ ] `LoadInventoryFromGameState` enforces a maximum resource count (e.g., 10,000 resources per load) to prevent memory exhaustion
- [ ] `GameState.AddResource` already rejects unknown types via `IsKnownType` (no change needed)
- [ ] Log spam from unknown resource warnings is eliminated (errors replace warnings)
- [ ] All callers of `StringToResourceType` are updated to handle the new error return
- [ ] Tests are updated to verify error behavior and size limits

## Steps

1. **Change `StringToResourceType` signature**
   - Open `internal/simulation/economy_integration.go`
   - Update function to return `(economy.ResourceType, error)`
   - For known legacy strings and valid economy types, return the type with `nil` error
   - For unknown strings, return `ResourceGrain` (or zero value) and an error describing the unknown type

2. **Update `ToEconomyResource`**
   - Handle the error from `StringToResourceType`; if error, return an invalid resource or propagate error (requires changing its signature as well)
   - Since `ToEconomyResource` is used in `LoadInventoryFromGameState`, we can change it to return `(economy.Resource, error)`

3. **Update `LoadInventoryFromGameState`**
   - Add a constant `maxResourcesPerLoad = 10000` (or a configurable limit)
   - Check `len(resources) > maxResourcesPerLoad` and return an error if exceeded
   - For each resource, call `ToEconomyResource`; if error, return it immediately
   - Remove the warning log (since error will be returned)

4. **Update `production.go` usage**
   - The single call to `StringToResourceType` in `production.go` must handle the error (log or propagate)

5. **Update tests**
   - `TestStringToResourceType_KnownLegacy` and `TestStringToResourceType_Unknown` need to check error returns
   - Add `TestLoadInventoryFromGameState_SizeLimit` to verify rejection of oversized arrays
   - Update any other tests that rely on the old behavior

6. **Run verification commands**
   - Build simulation package to ensure no compilation errors
   - Run integration tests to verify error propagation does not break existing functionality

## Code

Snippet for updated `StringToResourceType`:

```go
func StringToResourceType(s string) (economy.ResourceType, error) {
    rt := economy.ResourceType(s)
    if economy.IsValidType(rt) {
        return rt, nil
    }
    switch s {
    case "food":
        return economy.ResourceGrain, nil
    // ... all other legacy mappings
    default:
        return economy.ResourceGrain, fmt.Errorf("unknown resource type %q", s)
    }
}
```

Snippet for `LoadInventoryFromGameState` size limit:

```go
const maxResourcesPerLoad = 10000

func LoadInventoryFromGameState(inv *economy.Inventory, resources []Resource, defaultLocation string, produced economy.GameDate) error {
    if len(resources) > maxResourcesPerLoad {
        return fmt.Errorf("resource array too large: %d > %d", len(resources), maxResourcesPerLoad)
    }
    for _, sr := range resources {
        r, err := ToEconomyResource(sr)
        if err != nil {
            return fmt.Errorf("invalid resource %v: %w", sr, err)
        }
        r.Location = defaultLocation
        r.Produced = produced
        if err := inv.AddResource(defaultLocation, r); err != nil {
            return err
        }
    }
    return nil
}
```
