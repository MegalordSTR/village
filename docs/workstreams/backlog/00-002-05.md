# 00-002-05: Craftsmanship & Quality System

Feature: F002 (Resource Economy System)

## Goal

Implement resource quality tiers (Poor, Normal, Good, Excellent, Masterwork) and craftsmanship system where skilled workers produce higher quality goods. Quality affects tool durability, production speed, product value, and spoilage resistance. Create skill progression for craftsmen and quality inheritance from inputs to outputs.

## Scope Files

- `internal/economy/quality.go` - Quality tier definitions, quality effects, quality calculation utilities
- `internal/economy/craftsmanship.go` - Craftsmanship logic, skill-based quality determination, masterwork chances
- `internal/economy/skills.go` - Skill definitions for economic activities (farming, mining, smithing, weaving, baking)
- `internal/economy/quality_test.go` - Unit tests for quality calculations and craftsmanship

## Dependencies

- 00-002-01: Resource Definition System (resource quality field)
- 00-002-02: Production Chain System (quality inheritance in recipes)

## Acceptance Criteria

- [ ] QualityTier enum with numeric values (0=Poor, 4=Masterwork) and string representations
- [ ] Quality effects: tool durability (Poor: 0.5x, Masterwork: 3.0x), production speed modifiers, product value multipliers
- [ ] Spoilage resistance: higher quality reduces spoilage rate (Good: 0.7x, Masterwork: 0.3x)
- [ ] Craftsmanship: worker skill level determines achievable quality (novice: Poor-Normal, expert: Good-Excellent, master: Masterwork chance)
- [ ] Quality inheritance in production: output quality = weighted average of input quality + skill bonus
- [ ] Special materials enable higher quality (fine iron, seasoned wood)
- [ ] Skill progression: workers improve skills through practice, teaching possible
- [ ] Quality visible in UI (icons, tooltips planned for frontend workstream)
- [ ] Unit tests covering quality calculations and skill effects
- [ ] `go build ./internal/economy` passes
- [ ] `go test ./internal/economy -run TestQuality` passes with ≥80% coverage
