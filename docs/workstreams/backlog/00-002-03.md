# 00-002-03: Inventory & Storage System

Feature: F002 (Resource Economy System)

## Goal

Implement comprehensive inventory management with storage buildings, capacity limits, spoilage system for perishable goods, and stock tracking. Replace/extend the basic economic system with advanced inventory features: storage types (granary, warehouse, outdoor pile), spoilage rates based on storage conditions and quality, stock alerts, and resource transfer between locations.

## Scope Files

- `internal/economy/inventory.go` - Inventory management functions (Add, Remove, Query, Transfer)
- `internal/economy/storage.go` - Storage building definitions, capacity calculations, type-specific properties
- `internal/economy/spoilage.go` - Spoilage system with rates based on resource type, storage conditions, quality, age
- `internal/economy/inventory_test.go` - Unit tests for inventory operations and spoilage

## Dependencies

- 00-002-01: Resource Definition System (requires resource types and properties)

## Acceptance Criteria

- [ ] Inventory manager with methods: AddResource, RemoveResource, GetAvailable, TransferResource
- [ ] Storage building types: Granary (food, 1000kg), Warehouse (goods, 500 units), OutdoorPile (200 units, high spoilage)
- [ ] Capacity limits enforced per storage type
- [ ] Spoilage system: different rates per resource type (grain 1x, vegetables 2x, meat 3x, bread 0.5x)
- [ ] Spoilage affected by storage quality (0.5x for good storage, 2.0x for poor)
- [ ] Quality degradation when spoilage exceeds threshold
- [ ] Stock alerts when resources fall below configurable thresholds
- [ ] Resource aging tracking (production date)
- [ ] Integration with existing GameState.Resources (backward compatibility)
- [ ] Unit tests covering all inventory operations and spoilage scenarios
- [ ] `go build ./internal/economy` passes
- [ ] `go test ./internal/economy -run TestInventory` passes with ≥80% coverage
