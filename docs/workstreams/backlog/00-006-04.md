# 00-006-04: Update Development Tools and Security Audit

Feature: F006 (Go Modernization)

## Goal

Update development tool versions (Node.js, npm packages, Docker images) and perform security audit of dependencies. Ensure all tools are using current, secure versions.

## Scope Files

- `frontend/package.json` - Frontend dependencies
- `frontend/package-lock.json` - Locked dependency versions
- `.github/workflows/deploy.yml` - CI configuration for Node.js version
- `Dockerfile` - Base images for frontend and backend
- `docker-compose.yml` - Service definitions

## Dependencies

- Independent (can run in parallel with other Go modernization workstreams)

## Acceptance Criteria

- [ ] **Node.js Version**: CI workflows and Dockerfile use current LTS Node.js version (20.x)
- [ ] **npm Audit**: Run `npm audit` and fix critical/high vulnerabilities
- [ ] **npm Outdated**: Check for outdated packages and update where safe
- [ ] **Docker Base Images**: Use current Alpine versions for all Docker images
- [ ] **Security Scanning**: Add automated vulnerability scanning to CI (GitHub Code Scanning or Trivy)
- [ ] **Documentation**: Update README with security practices and update procedures

## Context

The project uses Node.js for frontend development and Docker for containerization. Keeping these tools updated ensures security, performance, and compatibility. The frontend likely has npm dependencies that need regular security updates. Docker base images should be regularly updated to include security patches.

## Input Files

1. `frontend/package.json` - Current npm dependencies
2. `.github/workflows/deploy.yml` - Current Node.js version in CI
3. `Dockerfile` - Current base images
4. `docker-compose.yml` - Service configurations

## Steps

1. **Check Node.js versions**
   - Verify `.github/workflows/deploy.yml` uses `node-version: '20'` (current LTS)
   - Check `Dockerfile` frontend builder uses `node:20-alpine`
   - Consider updating to latest 20.x minor version

2. **npm dependency audit**
   - Run `npm audit` in `frontend/` directory
   - Fix critical and high severity vulnerabilities
   - Run `npm outdated` to see outdated packages
   - Update packages cautiously, testing after each major update
   - Update `package-lock.json` after changes

3. **Docker base image updates**
   - Check `Dockerfile` backend uses `golang:1.25-alpine` (already current)
   - Check `Dockerfile` frontend uses `node:20-alpine` and `nginx:alpine`
   - Verify Alpine versions are current (no specific pinning)
   - Consider adding image vulnerability scanning step

4. **Add security scanning to CI**
   - Add GitHub Code Scanning or use Trivy action to scan for vulnerabilities
   - Configure to run on schedule and on PRs
   - Set up alerts for critical vulnerabilities

5. **Update documentation**
   - Add security section to README.md
   - Include instructions for updating dependencies
   - Document security scanning process

6. **Optional: Add dependabot configuration**
   - Create `.github/dependabot.yml` for automatic dependency updates
   - Configure for npm and Docker updates

## Code

### Example npm audit fix
```bash
cd frontend
npm audit fix --force
npm audit  # Verify fixes
```

### Example Dockerfile update
```dockerfile
# Ensure using current versions
FROM node:20-alpine AS frontend-builder
FROM nginx:alpine AS frontend
```

### Example GitHub Actions security scanning addition
```yaml
# Add to existing workflow or create new security.yml
- name: Run Trivy vulnerability scanner
  uses: aquasecurity/trivy-action@master
  with:
    scan-type: 'fs'
    scan-ref: '.'
    format: 'sarif'
    output: 'trivy-results.sarif'
```
