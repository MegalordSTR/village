# 00-006-06: Enforce Mandatory Linters and Fix Violations

Feature: F006 (Go Modernization)

## Goal

Enable all mandatory golangci-lint linters and ensure they only ignore violations in test files. Fix all errors and warnings reported by linters, removing unnecessary `//nolint` comments from production code.

## Scope Files

- `.golangci.yml` - Update configuration to enable mandatory linters
- `Makefile` - Ensure lint target uses updated configuration
- Production code files with linter violations (see list above)
- Test files with linter violations

## Dependencies

- WS-006-02 (Add Modern Go Linting Tools) - Linting infrastructure must be in place

## Acceptance Criteria

- [ ] `.golangci.yml` includes all mandatory linters: govet, staticcheck, errcheck, ineffassign, unused, misspell, gocyclo, gocognit, prealloc, gosec
- [ ] Linter exceptions are only for test files (no blanket exclusions for production code)
- [ ] `make lint` passes with zero issues after fixes
- [ ] All `//nolint` comments removed from production code (only allowed in test files)
- [ ] All errcheck violations fixed (error returns properly handled)
- [ ] All staticcheck violations fixed (deprecated APIs updated)
- [ ] All prealloc warnings addressed
- [ ] gosec warnings either fixed or properly justified with exclusions
- [ ] High cognitive complexity functions (gocognit) either refactored or justified
- [ ] All tests continue to pass (`go test ./...`)
- [ ] Project builds successfully (`go build ./...`)

## Context

The current golangci-lint configuration only enables govet, leaving many common issues undetected. Additionally, `//nolint` comments are used in production code to suppress warnings, which should be fixed instead. This workstream ensures code quality by enabling comprehensive linting and addressing all violations.

## Input Files

1. `.golangci.yml` - Current minimal configuration
2. `sdp/sdp-plugin/.golangci.yml` - Reference configuration with extensive linters
3. Output from `golangci-lint run` with extended linter set

## Steps

### 1. Update `.golangci.yml` configuration
- Enable mandatory linters: govet, staticcheck, errcheck, ineffassign, unused, misspell, gocyclo, gocognit, prealloc, gosec
- Configure linter settings:
  - gocyclo min-complexity: 15
  - gocognit min-complexity: 20
  - gosec exclusions: G204 (subprocess with variable) - acceptable for tests
- Set up exclusions only for test files:
  - Add `exclude-rules` for `*_test.go` files for appropriate linters (e.g., gocyclo, gocognit)
- Ensure no exclusions for production code paths

### 2. Fix errcheck violations in test files
- Review all errcheck violations (10 instances)
- For test files, check error returns and either:
  - Use `require.NoError` or `assert.NoError` from testing framework
  - Call `t.Fatal(err)` on error
  - Ignore with `//nolint:errcheck` if error is truly irrelevant (only in tests)

### 3. Fix staticcheck and prealloc warnings
- Replace deprecated `rand.Seed` with `rand.New(rand.NewSource(seed))` in `scarcity_test.go`
- Preallocate slices in `economy.go`, `environment.go`, `production.go` where appropriate

### 4. Address gosec warnings
- Evaluate each gosec warning (16 instances):
  - G104 (unhandled errors): Fix by handling errors in `main.go` and `inventory.go`
  - G404 (weak random): Add exclusions for simulation code where deterministic randomness is required
  - G115 (integer overflow): Review and add exclusions if safe
  - G204 (subprocess variable): Already excluded
- Update `.golangci.yml` gosec exclusions as needed

### 5. Remove unnecessary `//nolint` comments from production code
- Fix underlying issues instead of suppressing:
  - `cmd/village/main.go:13`: Handle error from `w.Write`
  - `cmd/village/main.go:31`: Consider adding error logging for `http.ListenAndServe`
  - `internal/economy/inventory.go:148`: Handle error from `AddResource`
- Remove `//nolint` comments after fixes

### 6. Refactor high cognitive complexity functions (optional stretch)
- `internal/economy/production_chains.go:14` - cognitive complexity 36
- `internal/simulation/environment_test.go:126` - cognitive complexity 35  
- `internal/simulation/production.go:316` - cognitive complexity 63
- Consider breaking down into smaller helper functions
- If refactoring is extensive, create follow-up workstream

### 7. Verify all fixes
- Run `make lint` to ensure zero issues
- Run `go test ./...` to ensure tests still pass
- Run `go build ./...` to ensure project builds

## Code

### Updated .golangci.yml example
```yaml
version: "2"
run:
  timeout: 5m
  modules-download-mode: readonly

linters:
  disable-all: true
  enable:
    - govet
    - staticcheck
    - errcheck
    - ineffassign
    - unused
    - misspell
    - gocyclo
    - gocognit
    - prealloc
    - gosec

issues:
  exclude-use-default: false
  max-issues-per-linter: 0
  max-same-issues: 0
  exclude-rules:
    - path: .*_test\.go$
      linters:
        - gocyclo
        - gocognit
        - errcheck

linters-settings:
  gosec:
    excludes:
      - G204  # subprocess launched with variable (acceptable in tests)
      - G404  # weak random number generator (deterministic simulation)
      - G115  # integer overflow (safe in context)
  gocyclo:
    min-complexity: 15
  gocognit:
    min-complexity: 20

output:
  format: colored-line-number
```
