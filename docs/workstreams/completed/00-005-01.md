# 00-005-01: Deployment Simplification - Remove Docker Registry Dependency

Feature: F005 (Deployment Simplification)

## Goal

Remove the dependency on Docker registry (ghcr.io) for deployment. Simplify the deployment process by building Docker images locally and immediately deploying them using docker-compose, either on the same CI runner or on a remote server via SSH. Eliminate the push/pull cycle to remote container registries.

## Scope Files

- `.github/workflows/deploy.yml` - Remove registry login, build-and-push job; add direct deployment job
- `scripts/deploy.sh` - Update to support CI/CD environment and potentially remote deployment
- `Makefile` - Update deploy target to reflect new process
- `Dockerfile` - Verify no registry dependencies remain
- `docker-compose.yml` - Ensure it uses locally built images
- `README.md` - Update deployment instructions
- `docs/deployment.md` - Create or update deployment documentation

## Dependencies

- None (independent infrastructure change)

## Acceptance Criteria

- [ ] Remove registry login and image push from `.github/workflows/deploy.yml`
- [ ] Update deploy job to build images locally and deploy using docker-compose
- [ ] Add SSH-based deployment option for remote servers (optional but recommended)
- [ ] Update `scripts/deploy.sh` to handle both local and CI/CD environments
- [ ] Ensure `Makefile deploy` target works with new process
- [ ] Verify `docker-compose.yml` uses `build:` directives rather than `image:` from registry
- [ ] Update documentation to reflect simplified deployment process
- [ ] Deployment can be triggered via GitHub Actions push to main branch
- [ ] No external dependencies on ghcr.io or other container registries
- [ ] `make deploy-dry-run` passes
- [ ] `bash scripts/deploy.sh local` works locally
- [ ] `docker-compose config` validates configuration
