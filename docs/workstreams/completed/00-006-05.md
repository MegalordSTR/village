# 00-006-05: Integration Tests and Build Verification

Feature: F006 (Go Modernization)

## Goal

Add comprehensive build verification and integration testing to ensure the project remains functional after tooling updates. Include version matrix testing, performance benchmarks, and race condition detection.

## Scope Files

- `.github/workflows/go.yml` - Extend with Go version matrix and integration tests
- `Makefile` - Add integration test targets
- `cmd/` - Command-line interface tests
- `internal/` - Integration tests for internal packages

- WS-006-01 (Update Go Version) - Need consistent Go version first
- WS-006-02 (Add Modern Go Linting Tools) - Linting should pass

## Acceptance Criteria

- [ ] **Go Version Matrix**: Test against multiple Go 1.25.x versions (e.g., 1.25.0, 1.25.7)
- [ ] **Race Detection**: All tests run with `-race` flag to detect data races
- [ ] **Integration Tests**: Add end-to-end integration tests for critical paths
- [ ] **Performance Benchmarks**: Add benchmarks for key simulation components

## Context

After updating Go version and tooling, comprehensive testing ensures no regressions are introduced. The simulation nature of the project requires deterministic behavior and performance characteristics. Race condition detection is critical for concurrent simulation code.

## Input Files

1. `.github/workflows/go.yml` - Existing test workflow
2. Existing test files in `internal/` and `cmd/`
3. Benchmark examples from Go standard library

## Steps

1. **Extend GitHub Actions workflow with Go version matrix**
   - Modify `.github/workflows/go.yml` to test multiple Go 1.25.x versions
   - Use `strategy.matrix` to define versions
   - Ensure tests pass on all versions

2. **Add race detection to CI**
   - Add `-race` flag to `go test` commands in CI
   - Ensure tests are race-condition safe
   - Fix any race conditions discovered

3. **Create integration test suite**
   - Identify critical integration points (simulation engine, database, API)
   - Create integration tests in `internal/integration/` or `tests/`
   - Use test fixtures and setup/teardown
   - Ensure tests are deterministic

4. **Add performance benchmarks**
   - Identify performance-critical components (simulation ticks, pathfinding, etc.)
   - Create benchmark tests using Go's `testing.B`
   - Add benchmark target to Makefile
   - Store benchmark results for comparison over time

5. **Build verification**
   - Add cross-compilation check for major platforms (linux/amd64, darwin/arm64)
   - Verify build with different build tags if any
   - Check that minimal binary size is maintained

6. **Update Makefile with new targets**
   - Add `test-integration` target
   - Add `bench` target for benchmarks
   - Add `test-race` target for race detection
   - Add `test-all` that runs all test types

7. **Verify test coverage**
   - Ensure test coverage remains acceptable (>80% for critical packages)
   - Update coverage reporting in CI if needed

## Code

### Go version matrix in GitHub Actions
```yaml
jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        go-version: ['1.25.0', '1.25.7']
    steps:
    - uses: actions/checkout@v4
    - name: Set up Go ${{ matrix.go-version }}
      uses: actions/setup-go@v5
      with:
        go-version: ${{ matrix.go-version }}
    - name: Test with race detection
      run: go test ./... -race
```

### Makefile additions
```makefile
.PHONY: test-integration bench test-race test-all

test-integration:
	go test ./internal/integration/... -v

bench:
	go test ./... -bench=. -benchmem

test-race:
	go test ./... -race

test-all: test test-race test-integration bench
```
