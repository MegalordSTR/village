# Release v0.1.1 - Go Modernization

**Release Date:** 2026-03-09  
**Tag:** v0.1.1  
**Previous Version:** v0.1.0

## Overview

This release introduces comprehensive Go modernization (F006) with tooling updates, security enhancements, and improved testing infrastructure. The updates ensure consistent Go version usage, add modern linting tools, perform a code audit for best practices, update development tools, and enhance integration testing.

## Features

### 1. Update Go Version and CI Configuration (00-006-01)
- Consistent Go 1.25.7 across CI/CD workflows, Docker, and go.mod
- Updated `.github/workflows/go.yml` to use `go-version: '1.25'`
- Verified `Dockerfile` uses `golang:1.25-alpine`
- Added Go version check target to Makefile
- Ensured all tests pass with updated configuration

### 2. Add Modern Go Linting Tools (00-006-02)
- Integrated `golangci-lint` with comprehensive rule set
- Created `.golangci.yml` configuration with staticcheck, govet, gosec, errcheck, etc.
- Added `lint` and `lint-fix` targets to Makefile
- Created GitHub Actions lint workflow (`.github/workflows/lint.yml`)
- Updated README.md with linting instructions

### 3. Code Audit for Modern Go Best Practices (00-006-03)
- Comprehensive analysis of 801 Go files for compliance with Go 1.18+ practices
- Error handling analysis: 259 `fmt.Errorf` calls without `%w` wrapping identified
- Generics opportunities: 323 `interface{}` usage patterns cataloged
- Concurrency patterns audit: goroutine usage, synchronization, channel patterns
- API design review: package boundaries, interface design, exported identifiers
- Standard library usage: deprecated functions, newer packages (slices, maps, cmp)
- Performance patterns: allocation hotspots, inefficient loops
- Generated prioritized audit report with high/medium/low recommendations

### 4. Update Development Tools and Security Audit (00-006-04)
- Updated Node.js to LTS version 20.x in CI and Dockerfile
- Ran `npm audit` and fixed critical/high vulnerabilities
- Updated npm dependencies to current secure versions
- Ensured Docker base images use current Alpine versions
- Added automated vulnerability scanning to CI (Trivy)
- Enhanced security documentation in README

### 5. Integration Tests and Build Verification (00-006-05)
- Extended GitHub Actions workflow with Go version matrix testing (1.25.0, 1.25.7)
- Added race detection (`-race` flag) to all test runs
- Created integration test suite for critical paths
- Added performance benchmarks for simulation components
- Build verification for multiple OS/arch combinations
- Updated Makefile with `test-integration`, `bench`, `test-race`, `test-all` targets

No migration required. The updates are transparent to existing functionality.

## Known Issues

- Test coverage (59.3%) below target threshold of 80% (existing issue, not introduced by this release)
- Cyclomatic complexity violations in some legacy code (existing issue)
- Some error wrapping opportunities identified in audit not yet implemented (planned for future work)

## Deployment Instructions

1. Ensure local Go version is 1.25.x (`go version`)
2. Run `make lint` to verify linting passes
3. Run `make test-all` to execute full test suite (unit, integration, race detection, benchmarks)
4. Run `make build` to verify compilation
5. Deploy using existing methods:
   - Local: `docker-compose up --build`
   - Production: `./scripts/deploy.sh production`

The updated CI/CD pipeline will automatically run enhanced checks on all future commits.
